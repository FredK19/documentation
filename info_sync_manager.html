Sync Manager Troubleshooting

Overview

If you are getting a database not found error on a new machine or even an old machine
This should not be done on a regular basis!!

Error: Database could not be found

If you go to sign into Order Manager 8.x and get a pop-up that says database cannot be found, here are the follow things you should ask yourself before proceeding.
  • Has a new user been added?
  • Was a password changed recently?
  • Did I install the correct software?
  • When was the last time I logged in successfully on this machine to ZID?
  • Am I typing my username with the correct case? (case sensitive)
    After answering these questions there are two routes that you can take next.
      Route 1: Open a support ticket to have a tech log in to your machine and troubleshoot the error. When opening this ticket be sure to include the answer to the question above.
      This would be billable under geek services Doc 50551. This could be cheaper than doing it yourself and entering incorrect information.

      Route 2:Troubleshoot the issue yourself by checking the software versions and/or if a password was created/changed, you will need to do a nuke config. Use caution when doing this. You will need admin password and database password.
      If you are unsure about the nuke config you should probably take route 1.

      If you decide to take route 2 and an error occurs due to taking route 2 it will be billable as *esop* time! Refer to Doc 50551 webdoc for pricing on services
