Guide to Custom HTML Wizards

Overview

An HTML wizard can include information from the merchants profile, from the product being used, and from the channel it is being created in.  HTML Wizards typically combine data like the product name, description, along with merchant shipping policy, and payment details. 

The Zoovy Product Manager comes pre-packaged with many HTML Wizards which are classified as either SIMPLE or ADVANCED. For the merchants that have special requirements, or just want their very own look, Zoovy provides a method for creating custom HTML Wizards.

NOTE: If you remove the email address from any of the pre-formatted HTML Wizards, the system will remove your customer service email address from your company information. Please select an HTML Wizard that does not contain a field for your email address if you do not wish to include your email in the auction.

Components

Each HTML Wizard has three files:
  • A TXT file - (called the definition file) which describes the data that should be provided to complete the HTML Wizard.
  • A HTML File - contains the HTML content which will be interpolated with the data that is entered from the definition file.
  • A Thumbnail (optional) - used to provide a preview of the HTML Wizard for users.

    • Definition

    GREYBOX1.txt is the list of information needed by the HTMLWIZ that is then combined with the contents of GREYBOX1.html to create the final HTML for your listing. You can edit the GREYBOX1.txt and change the list of information needed, for example you could remove it so it does not prompt you for your logo, payment policy, return policy, etc. and then you could edit GREYBOX1.html and design it so payment policy, return policy, etc. are not part of the auction. In the GREYBOX1.txt you might find the following tag:
    <input TYPE="TEXTBOX" NAME="merchant:zoovy:prod_desc" PROMPT="Product Description" SUB="FOO"></input>
    The TYPE= parameter tells it what type of input element you are dealing with, valid types are TEXTBOX, TEXTAREA, and IMAGE The NAME= tells the product manager which data attribute to load the data from, obtain a list of commonly used field names, or create your own. In the following example replace yourname with your username, then somename with a valid tag name which can be any combination of letters or numbers. You can create global fields by going: merchant:yourname:somename , or fields which are associated with a product by using the format: product:yourname:somename The PROMPT field is what is displayed to you, the user of the HTML Wizard that tells you want to put in that field. Finally, the SUB= field tells the HTML Wizard to substitute the contents of what you entered into the GREYBOX1.html file.

    HTML File

    In the GREYBOX1.html file you would have the following code:
    <!-- FOO -->
    would be replaced with whatever data was entered into the
    <input SUB="FOO" .. > tag.
    The name isn't important, just make sure whatever is inside the data of the SUB= also appears inside an HTML comment in the .html file.

    Supported Properties

    Here is a table of all supported INPUT TYPES and their respective properties.
    INPUT TYPE PROPERTIES SUB BEHAVIOR
    (any input type) PROMPT - the prompt displayed to the user. e.g. "Enter Something Here"
    SUB - the name of the variable to be replaced in the text. (read the SUB VALUES) in the adjacent columns for more info how substution works.
    TYPE - specifies the input type (choose from the values below) e.g. INPUT="TEXTBOX" NAME - the namespace:owner:attribute to save/load data from.
    LOADFROM - the default value to load from if NAME= is not populated, in namespace:owner:attribute. Can accept a comma separated list of values
    READONLY - values are 0 or 1 makes a field non-editable, this is especially useful if you are referencing data on the parent form, such as channel:ebay:title, or do not want to allow the user to accidentally edit the information e.g. product:zoovy:prod_name
    VALUE - the default value, if nothing can be found in NAME, or LOADFROM values.
    PRETEXT - a text string which is prepended to the input if the data is not a blank.
    POSTTEXT - a text string which is appended to the input string if the data is not a blank.
    URLENCODE - if 1 then performs url encoding (& becomes %37), encodes all non-alpha numeric characters. (used for passing data to another program via an HTTP GET)
    ENTITYENCODE - if 1 then performs basic entity encoding (& becomes &amp;, < becomes &lt;, > becomes &gt; " becomes &quot;) handy for escaping forms inside forms, etc.

    SELECT LIST - references a >LIST< element, uses the same format as the channel definition input type.
    IMAGE ZOOM - values are 0 or 1; if 1 the SUBNAME image tag zoomable, effectively this places a href tag around the <IMG> tag which links to the source image url (RAWURL)
    BORDER - values are 0 or greater; determines of the image has a border around it (only appears if the image is Zoomable) SUFFIX - when an image is imported it is automatically renamed to the SKU, SUFFIX should be used if more than one image file is being imported. e.g. SUFFIX="-2" and a SKU of ABC would produce an image named ABC-2.jpg.     Reserved Substitution Names

    The following variables are always available, and should be considered reserved:
    The username of the currently logged in user.
    The SKU of the product which is currently in context.

    Shipping Calculator

    Accounts with the EBAY bundle can include a link to the shipping calculator by simply linking to:
    <a target="_blank" href="http://<!-- USERNAME -->.zoovy.com/shipquote.cgis?product=0%2A">Get a Shipping Quote</a>
    You can also allow the user to enter a zip code, or other information. Refer to the Shipping Calculator Developer Guide for more information.
