Advanced Order Import Guide

Overview

Please note: This is an advanced feature and requires that you first schedule training before you submit a technical support ticket. If you are a new Zoovy merchant and wish to use this feature, please schedule training.

Place one attribute in the following format in each column of the header to denote which field you will be saving, the import process is case sensitive. Also remember that spaces are not allowed between each header column.
Example CSV File:
#ALLOW_CRLF #GROUP_BY=%EMAIL #STATUS=COMPLETED #DECREMENT_INV=Y %EMAIL,%FULLNAME,bill_address1,bill_city,bill_state,bill_zip, bill_phone,%SKU,%QTY,%DESCRIPTION,%PRICE,payment_method, order_notes,%SHIPPING_COST homer@simpsons.com,"Joseph Chou","123 Some Street",Springfield,MG,12345, 555-555-5680,TEST,20, "This is the test description",9.84,"IMPORT", "Order Notes for this order",6.99

The example above would create one customer record for customer Homer Simpson with billing contact information. Yes, we know "MG" is not a state, but do you know where the Springfield of the Simpson's is? Bonus question, what does "MG" indicate?

Column Headings

%EMAIL Customer's email address, this is the unique identifier for the customer record. This should be the first column in your CSV file.
%FULLNAME Customer's full name (first and last)
bill_company Billing company name (if applicable)
bill_fullname Billing first and last name
bill_phone Billing phone number
bill_address1 Billing street address
bill_address2 Billing additional address
bill_city Billing city
bill_state Billing state
bill_zip Billing Zip Code
bill_country Billing Country
ship_company Company Name (if applicable)
ship_fullname Shipping first and last name
ship_phone Shipping phone number
ship_address1 Shipping street address
ship_address2 Shipping  additional address
ship_city Shipping city
ship_state Shipping state
ship_zip Shipping Zip Code (Domestic)
ship_int_zip Shipping Zip Code (International)
ship_province Shipping Province (International)
ship_country Shipping Country
%PASSWORD Customer's login password (importing with a value of * in the column will automatically generate a password)
%SCHEDULE Schedule Code
%NEWSLETTER If you are using the newsletter tool and have multiple lists, import is configured as follows:
  Not subscribed - 0
List 1 - 1
List 2 - 2
List 3 - 4
List 4 - 8
List 5 - 16
List 6 - 32
List 7 - 64
List 8 - 128
List 9 - 256

If you want to add the customer to list #1, set the value to 1
If you want to add the customer to list #5, set the value to 16
if you want to add them to list #1, #4 and #5 then you'd do the following: 1 + 8 + 16 = 25

%SKU SKU sold
%QTY Quantity of the SKU sold
%DESCRIPTION SKU description
%PRICE SKU price
%SHIPPING_COST Shipping Cost
order_notes Notes for the Order
payment_method Payment Method: CREDIT, AMAZON, PAYPAL, CASH, OTHER
payment_status Payment Status: 000=PAID, 010=PAID via AMAZON.com, 050=PAID via PAYPAL, 100=PENDING, 200=DENIED, 300=CANCELLED

Optional Headers - these values affect _all_ orders in the import

#GROUP_BY bill_email, bill_phone, %EMAIL, %ORDERID
Automatically group lines of the import into one order vs one order per line (default => blank)
#STATUS RECENT, PENDING, APPROVED, COMPLETED, DELETED, ARCHIVE, BACKORDER
Set the status of the orders (default=> RECENT)
#DECREMENT_INV Y
Decrement your inventory (default => N)
#ORDER_CREATE_EMAIL Y or PROFILE name
Send an Order Create email to your customers (default=> N)
#ORDER_NOTES Order notes
#PAYMENT_METHOD Payment Method: CREDIT, AMAZON, PAYPAL, CASH, OTHER
#PAYMENT_STATUS Payment Status: 000=PAID, 010=PAID via AMAZON.com, 050=PAID via PAYPAL, 100=PENDING, 200=DENIED, 300=CANCELLED
The %NEWSLETTER field can NOT be changed in the for existing customers by CSV import. %NEWSLETTER will only be added to new customer records being created.

Frequently Asked Questions

Q: Why can't I change the value for %NEWSLETTER when updating existing customers?
A: Because the customer must have chosen to receive email from you, which they can do in their account management area. Any email sent without their direct consent would be spam and is strictly against Zoovy Terms of Service (TOS) as well as all applicable anti-spam laws.
