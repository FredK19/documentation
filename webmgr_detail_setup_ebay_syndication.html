eBay Syndication Feed Setup

Overview

Syndicated ebay listings are an automated way to establish a steady long term presence for product listings, which will lead to improved search rankings using the ebay algorithm. This feature allows products to list(and re-list)automatically to ebay, and any sales from those listings to be processed through the Zoovy order system.

Syndicated listings work differently from manually created listings.
  • 1. All available inventory is displayed for syndicated listings. The number available displayed in the ebay listing is updated as product inventory is edited in the Zoovy interface (usually within ten minutes). If inventory reaches zero, the ebay listing will be ended.
  • 2. Inventory reserves are not created for syndicated listings. This allows merchants with limited inventory to list their products to multiple marketplaces.
  • Note: Listing products with limited inventory can lead to overselling, especially when the product is a "fast mover". The merchant accepts all responsibility should a product oversell due to limited inventory, and over exposure.
  • 3. Changes made to product or listing configuration are automatically applied to existing ebay listings (usually within ten minutes. There is no way (or need) to Refresh syndicated listings.
  • 4. Products with up to two inventoriable options can be listed to ebay ONLY by using ebay syndication. Other listing methods do not support product options.
  • 5. ALL syndicated ebay listings are Fixed Price, and GTC (Good 'Til Cancelled).
    Enabling ebay syndication at the product level will remove the Fixed Price configuration from the ebay 2.0 panel. Conversely, disabling syndication at the product level will remove the Syndication Test feature from the ebay 2.0 panel.
    This is to protect against the accidental creation of duplicate fixed price listings, which is in violation of ebay regulations, and can lead to disciplinary action.
    Auction type listings can be created either manually using Product Editor, or with automatic re-list functionality using the Powerlister feature.

    • Requirements

  • 1. The Auction bundle. An account must have Auction bundle functionality. Setup >> Add/Remove Features >> Auction Bundle [Add],
  • 2. An ebay account / Token must be configured / enabled. Setup >> Ebay Configuration.
  • 3. At least one ebay Profile must be created (both Config & Shipping).
  • 4. Ebay Syndication must be enabled at the Syndication level. Syndication >> Ebay.
  • 5. Ebay syndication must be enabled at the Product level. This can be done by checking the box for Ebay Syndication in the Syndication Panel of Product Editor, or by setting the ebay:ts product attribute to a value of 1, using either the Product Power Tool or a CSV import. (setting the ebay:ts attribute to 0 disables ebay syndication for that product).
  • 6. Products must reference a valid ebay profile and ebay category. Both of these can be selected in the ebay 2.0 panel of Product Editor.
    If you have an ebaystore, and an ebaystore category is configured at the product level, the syndicated listing will also be displayed in your ebaystore.

    • Ebay Syndication Configuration

    Edit Partition:
    The Edit Partition sub-tab is where you will view the current status of your syndication and configure some of the basic syndication settings.

    Activating Syndication
    If your syndication status is inactive, just click the 'save' button to activate it. Clicking save again should bring up a Status message like the one in the window below.

    If your syndication is inactive for any reason, the status bar will show the following:

    Status: Inactive - no feeds will be transmitted.

    When your syndication is active and running normally, you should see something like this:

    Status:

    Last INVENTORY Update: Fri 18-Nov-2011

    Last Diagnostic Message:
    Last Action: inventory at 20111118 06:31:46
    Inventory 12992 at 20111118 06:31:46
    Submit New Products:
    Checking this box enables ebay syndication for your account.

    Use AutoPilot on New Items:
    Enabling this feature allows the system to make "best guess" decisions when a required product field has not been defined.
    Example: If AutoPilot is enabled, and the Fixed Price field for a product is left undefined, the system will use the zoovy:base_price amount for the ebay listing. If Autopilot were not enabled in this scenario, the listing attempt would generate an error, and the product would fail to list.

    Detailed Logging:
    This feature should only be enabled if instructed to do so by Zoovy Technical Support. This feature implements extensive logging of ebay listing events which are used to trouble shoot possible system, or configuration issues.
    Enabling Detailed Logging is NOT recommended, as the amount of data collected by this feature is extensive, and can easily generate additional charges for the merchant due to the amount of disc space required to store so much additional data.

    Ebay Syndication for Products with Options

    ONLY inventoriable product options are valid for ebay syndication.

  • 1. A product can have up to two inventoriable option groups display on a syndicated ebay listing. For example, you could have a shirt display with color and size options, or just size options, but not color, size *and* style options at the same time. All options with available inventory will be displayed in the ebay listing.
  • 2. A listing will stay active as long as at least one of the SKUs has available inventory.
  • 3. As available inventory for specific options is adjusted in Zoovy, that change will be reflected on the ebay listing (usually within ten minutes). If a specific option reaches zero inventory, that option will no longer display in the listing, but options with positive inventory will continue to be available to shoppers.

    • Ebay Syndication Tools

    ebay categories:
    This is where merchants who have an ebaystore can map their ebaystore categories to valid ebay categories.

    Webdoc:
    A link to this webdoc.

    History:
    Here you can see a timeline of some basic data about your syndication feeds to Ebay. The History sub-tab Has information on:

    -DATE: The date that the syndication event took place.
    -TYPE: The type of syndication event.
    -TOTAL: How many products you have (account wide) that are set for syndication based on ebay:ts setting.
    -VALIDATED: How many products you have (account wide) that are set for syndication based on ebay:ts setting.
    -TRANSMITTED: How many products you have still ready to go after removing products with zero inventory, suspended products, products already on eBay, and products that are configured incorrectly and will get eBay errors.

    Feed Errors:
    a readout of your most recent feed errors on individual products. Use this sub-tab to correct your product errors and get the products back onto Ebay. For a list of common Ebay errors and how to fix them, go to Doc 50752


    Diagnostics:
    This tool will show you VERY detailed information about your syndication feed. You can enter one product ID to diagnose a single product, or leave the field blank to diagnose your whole store. Some of the checks that the diagnostics tool performs are:

    -How many products you have (account wide) that are set for syndication based on ebay:ts setting.
    -Products with zero inventory.
    -Products that are suspended from eBay syndication.
    -Filter Launch: Products already on eBay.
    -Validation: Products that are configured incorrectly and will get eBay errors.

    Ebay Tools in Product Editor

    Syndication Test: This will run a test syndication for that product, and display any errors or warning detected during the test.
    Warnings will generally not prevent a listing.
    Errors will prevent a product from listing successfully.

    Listing Events: The Listing Events panel will show all attempts to create a listing for a product. Failed attempts will display the error that prevented the listing, while successful attempts will display the resulting live listing.
