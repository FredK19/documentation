Order Manager (ZOM)- Getting Started with version 6.x

Overview

This document covers procedures to create new user, login, and get Order Manager setup for initial use on version 6.012 and above.

Steps required in order to log into Order Manager (ZOM)

NEVER modify an order online after you have synchronized that order with Zoovy Order Manager.
First you will need to create a manage user. To create a manage user please follow the following steps.
  • Step 1. Log into Zoovy.com
  • Step 2. Click on Setup > Manage user log in
  • Step 3. Create a new user
  • Step 4. Create a password for the user
  • Step 5. Click save
    Please note the manage users are not restricted when using ZOM. This behavior will change in the future. When creating the manage user, the rights the user has will be referenced in later versions of ZOM.
    • Many people get confused by what is displayed as the "username" in the web interface and what the actual username you should be entering is. While the display shows the username to be "companyname*username", the actual username you will use to log into Order Manager is simply the "username" portion, you will omit the "companyname*" portion.
  • SOHO - The Small Office/Home Office version stores your company database file on the local hard disk. The database file can be shared with SOHO Clients via a shared network drive.
  • SOHO Client - Computers configured as a SOHO Client may not synchronize data, they must use a database created by a SOHO installation. To avoid performance issues you should never have more than 3 SOHO Clients computers + 1 SOHO Master in use at the same time (instead use Enterprise).
  • Enteprise Client - to use this option you must have an Enterprise server on your network. Enterprise servers automatically synchronize data with Zoovy, and offer significant functionality and performance improvements for companies with large databases.
    • You may only have ONE active database per account (SOHO or Enterprise). Having multiple databases is NOT supported and will cause data corruption. Any attempt to install additional databases will disable all existing databases.
    Logging into ZOM v6.xxx
  • Step 1. Click on Admin > Database Config
  • Step 2. Click on the SOHO tab
  • Step 3. Input the Admin username and password (Zoovy username and password)
  • Step 4. Keep the box checked for FIND AND UPGRADE PRE V6.010 DATABASE (RECOMMENDED)
  • Step 5. Click SAVE
  • Step 6. Input the manage user login name that you created in the text box Username (the name only with no *)
  • Step 7. Input the password for the manage user that you created in the text box Password
  • Step 8. Click Login
  • Step 9. Click Sync
    • If you change the Admin password to log into to zoovy.com, you will also need to change the Admin password within ZOM. To change the Admin password in ZOM please follow the following steps:
  • Step 1. Open ZOM
  • Step 2. Click on Admin > Account Security
  • Step 3. Input the Admin username
  • Step 4. Input the new Admin password
  • Step 5. Click update
    • If you are seeing the following error:
    #Z2000: API ERROR: Security Digests do not match - probably invalid password.
    You can address this issue by reauthorizing the security tokens as outlined in the HINT above.

    What to do after Login

    There are several actions that you should take after login to get your data into the database and ready for use:
  • Full standard syncronization - this can be performed by clicking on the "Sync" button in the menu bar at the top of the screen.
  • Company Information syncronization - this is accessed from Admin > Company > Company Syncronization
  • Product syncronization - accessed from Admin > Download Products from Website
    In order to get your products to show in the "Add Item" window, be sure to check the "Show All " checkbox.
  • Setup Email Preferences - Admin > Setup > Email Preferences
  • Finally you can take time at this stage if you desire to setup any 3rd party system integration that you may use for shipping (i.e. FedEx, UPS, Endicia) Once you have done the steps above you should find your ZOM database populated with the data you need and ready for use. Also reference wedboc Doc 50427] for additional information.
