Worldpac Guide - DEPRECATED

Overview

The Worldpac functionality is no longer offered by Zoovy, all sites which integrated this technology have long since gone out of business and created virtually zero sales during their tenure. Despite the best of intentions we have found that Worldpac customers have experienced a much lower than anticipated level of success with the Worldpac platform. We will consider future Worldpac integrations on a case by case basis, although it is very likely we will not integrate this into future sites.
Although there are many reason for this, we believe these outline the primary reasons for this decision:
  • Too much competition, Worldpac pricing not competitive.
  • Zoovy site wrapper and layouts must be "crippled" to be Worldpac compatible, this inevitably hurts sales of non-Worldpac products.
  • Worldpac is not compatible with the built-in cross selling, search, or product option features of Zoovy, and further requires that they must all be disabled and/or not used. Worldpac does not provide it's own equivalent.
  • Many Zoovy features such as Amazon, eBay, Googlebase, Google sitemaps, etc. are not compatible with Worldpac because of the way Worldpac require integration.
  • Zoovy hosted products do not appear in the worldpac catalog.
  • The worldpac database has millions of SKU's, but those SKU's are not visible by Zoovy, it is not possible to import the Worldpac database (they don't even offer it).
  • It is very difficult (expensive) to program the worldpac JSP (java server pages) to integrate with the Zoovy platform. Keep those changes up to date is very hard, and debugging is tedious and very difficult to accurately predict the hours required. Customers do not want to give an "open" budget, but it is not possible to provide a reliable fixed price budget due to unforseen circumstances which seem to arise in each configuration.
  • Zoovy is developing it's own catalog application which does not have these limitations which we may selectively release in the future. Some vertical industries are already supported, please consult the Zoovy sales team.

    • About Worldpac

    WORLDPAC imports, distributes and delivers automotive original equipment quality replacement parts for the import repair specialty market. They offer the import specialist 'one source' for over 20 import car makes with 60,000+ specific parts and over one million unique catalog applications. The Worldpac database lookup can be integrated with a Zoovy site in a very "loose" way. You can learn more at http://www.worldpac.com.

    Configuration and Installation

    The prerequisite to getting going is a username and password which must be assigned by WorldPac. You typically have to pay some sort of membership fee (around $1,000). The installation steps are pretty simple:
  • You must build or select a Zoovy wrapper.
  • You go into developer and click "enable soft cart" - and input "ANY" into the Allowed Referrers list.
  • You must install the Zoovy wrapper into the worldpac site via FTP.
  • You customize the worldpac site so submits products to the Zoovy cart.
  • You must create categories for each brand you want to carry
  • You then paste HTML into the each category which links to the worldpac site.
  • Thats it, you're done.

    Bugs and Caveats

  • Worldpac requires a popup, and does not work on browsers which block popups.
  • Worldpac parts are *NOT* Zoovy products, they are hosted by Worldpac. They cannot be put into product lists, intelligently upsold, launched to eBay, or anything like that - they are simply there for cross selling.
  • Zoovy does not support automatic ordering. If you receive orders for worldpac, you can order them via the phone, fax or via their SpeedDial application.
    • This integration is not part of the standard Zoovy offering. A minimum budget / spending commitment of $15,000 in custom development is necessary to begin discussions. All developer time spent during the planning phase is considered billable (sorry - no free quotes).
