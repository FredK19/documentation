Quickbooks Error 2147220472

Issue

When trying to export an order to Quickbooks, I get an error:
"Could not start Quickbooks,--2147220472"
Please note
This is not a Zoovy error. This is an error Intuit recently explained as an issue with recent Windows updates. This error has been seen to effect both XP and Vista Systems, and is now know to be an issue with .NET framework. Zoovy can attempt to troubleshoot this issue for you, but the service is billable as NZPS (Non-Zoovy Problem Solving). For information regarding billing rates for this issue, please refer to this webdoc: Doc 50551
The instructions below are a suggested method of correction have not not be proven to work on all systems. It is recommended that you contact Intuit support in regards to this issue and reference the error code provided.
Intuit Phone Support: 1-800-428-3170

Before you Proceed

This Correction process has potential for causing issues with other programs, or possibly your operating system if not performed correctly. Please read the process first, and should you not feel comfortable, contact our Tech support line and a representative can assist you.

Uninstall

In this process, you are going to uninstall all related files.
  • Windows XP users will click on Start, Control Panel, Add/ Remove programs.
  • Windows Vista users Click Start, Computer, Add/Remove Programs.
    • -Please remove the following:
  • All versions of Zoovy Integrated Desktop, Zoovy Order Manager, Zoovy Warehouse manager
  • Quickbooks
  • Quickbooks Patches (QBXMLRP2, QBFC4_0)
  • Microsoft .NET Framework (All Versions)

    Further Deletion

    Depending on the size of your Database, you may choose to either Delete the remaining Zoovy folders, or relocate your Database Folder to save initial Sync time. If your Syncing time has seen an hour plus, I strongly suggest relocating, or you may also choose to do this during company down time. PLEASE READ THIS ENTIRE SECTION ONCE BEFORE FOLLOWING.
    -XP users go to C:\Program Files\ and delete the Zoovy Folder
    XP Database Relocating Go to C:\Program Files\Zoovy\Desktop\ And move the folder titled your company Name to your desktop. Delete the rest of the Zoovy folder.
    -Vista Users Go to C:\Program Files\ and delete the Zoovy Folder
    Vista Database Relocating If you have your UAC ON, Zoovy will actually install in 2 different places. The Program Files, as posted above, and a virtual drive created by Windows. If off, your company database file will simply be in C:\Program Files\Zoovy\Desktop\.
    Windows Vista users must have the UAC turned ON and left on in order to prevent issues with the integration.
    The virtual drive Zoovy database location is as follows:
    C:\Users\(User Name)\Appdata\local\virtual store\program files\Zoovy\desktop\(Account name)
    If you do not see the Appdata folder, you have hidden folders enabled. In a folder window, click on Organize, Folders and search options, View, and Toggle to make hidden folders visible
    Create a new folder on the desktop and Call it "Z1"(or something distinguishing). Relocate the company folder from the virtual directory to this new folder. Delete the rest of the Zoovy Folder, as well as the one in the standard Program files directory.

    .NET Further Deletion

    Follow this link to download the .NET clean up tool

    Install the Tool and run. You are going to REMOVE ALL VERSIONS of .NET from your system. The program will tell you when it is complete. Close when done, Restart your computer.
    • Please note your system activities during the restart. Keep watch for any unusual events or errors. If any occur, please note in detail the issue and contact us.

    Reinstall

    Please follow the links in the Downloads section of your Zoovy Account for the latest release of Microsoft .NET Framework and ZID to your desktop.

    Installation process

    Vista Users Please be sure you have the UAC (User Access Control) Enabled. Check this by going into your control panel, clicking on User accounts, and you will see the toggle link for the UAC.
    Aside from the agreement, .NET Should install straight forward. It should not require a reboot.
    Install ZID next. Once the installation is complete, you will be prompted to set up SOHO. Click the SOHO tab, and log in here using your Zoovy login information. Once complete, it will leave you at the login screen. DO NOT LOGIN YET. Click on the Admin tab, and log in here using your Zoovy login information as well.
    If you deleted your initial Database Folders You should simply be able to login to ZID and sync. Be sure you do a company sync, and enable your email as you may have done in your previous install.
    If you saved your initial Database folders Please close ZID, and move your company file from the "Z1" folder back into the same virtual directory as before, overwriting any folder created under the same name. Logging into Zoovy, you should see your orders if done correctly.
    Once this is confirmed to work right, you should be fine to install Quickbooks. Assuming you have had training in the integration of Quickbooks, after the install, you should be able to map Zoovy to your QB company file and configure Quickbooks to allow the integration. If you need assistance with this, please set up an implementation meeting.
    Try to export an order to Quickbooks, the error should be corrected now.
