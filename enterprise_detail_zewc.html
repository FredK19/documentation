Installation

AppForge Booster

Before installing the ZEWC client you must have Appforge Booster+ installed on your Symbol 1846. Booster is a runtime environment which allows the ZEWC client to interface with the Symbol scanner. You will need to purchase a copy of the Booster software $25.00, or download a 30 day evaluation copy.
The version needed is the AppForge Booster Plus for the Symbol SPT 1800 (Version 3.5, Build 633 [or better]) Make sure you buy a license for the Symbol SPT1800 version, not just the PalmOS version.
Once you have installed Booster on your PC, then HotSync your palm pilot and the Booster application should be loaded. The first time Booster is HotSynced, it can take up to 5 minute to transfer all the necessary libraries. Once Booster been been successfully installed on the Palm Pilot you should verify that it is registered properly with the appropriate registration code that you were given by AppForge. You can verify this by going to the Booster application on the Palm.
After Booster has been installed - follow the steps below to install the ZEWC software.

Installing the PRC file

  • Download the newest version of the ZEWC-Install.prc file from the downloads area on your server.
  • Go to Start / Programs / Symbol SPT Desktop and run the installation tool.
  • Use the installation tool to locate and add the file to the Palm.
  • Press the Hotsync button on your Palm to download the software.

    Config File

    The configuration for ZEWC is done via the memo feature of the PalmOS. PalmOS automatically names a file by whatever appears in the first line. ZEWC looks for a file which is named "zewc" (without the quotes).

    The format for values are key=value - with each key and value pair on their own line. The following are a list of properties which can be configured in the memo file.
    KEY VALUE NOTES:
    ip x.x.x.x Notes: can be either a DNS name, or an ip address of the Zoovy server. Using an IP address is preferable to increase speed and reduce network traffic and/or possible errors.
    merchant username the zoovy username of the store this client will connect to.
    offline sets the scanner to "offline" mode where no wireless network is avaialble, but you still want to scan and store bar codes.
    eof signifies the end of file, any values below this line will be ignored. (it just lets us know that got to the end of the file)

    Example zewc config file

    zewc ip=192.168.1.100 merchant=username eof=
    To enable offline mode for the wireless client, add the following line to the config file: zewc ip=192.168.1.100 merchant=username offline= eof= The offline statement must be entered before "eof"

    Usage

    Along the bottom of the SPT1846 you will see four grey buttons, the outer left and right buttons toggle between modes. By default the client will assume that it is in an environment with an "always on" wireless network. A list of modes is given below.

    Store

    Store simply lets you scan an item, and a location and transmits that data to the server.

    Multi-Store

    Multi-store lets you scan one or more items, then a location. The toggle buttons along the top of the scanner can be used to move between the fields, pressing the downward toggle button twice will cause the batch to be submitted.

    Locate

    Locate lets you scan an invoice or packing slip, and query the server, and display results where the items on the invoice are.

    Multi-Locate

    Multi-locate lets you scan one or more invoices and/or packing slips and then retrieve a combined pick list for each of the items.

    Verify

    The verify mode is used to simply verify that a SKU is in the correct position.
