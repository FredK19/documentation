Issue #161254: Paypal charging wrong shipping amount

Order's going through Paypal have incorrect shipping amount (does not match invoice).
This is *NOT* a Zoovy issue. This is a *PAYPAL* issue.
We recommend customers contact Paypal's support group regarding this issue. (We hope that *SOMEDAY* Paypal might actually correct it if they get enough merchants who are confused by it.)
Zoovy support staff will not be able to correct this issue without you providing your Paypal login and password.

Resolution

The sellers Paypal account is incorrectly configured to add shipping to every order, even though Zoovy CORRECTLY passes the shipping to Paypal.
This is ridiculously easy to do since eBay helps sellers configure this.

Steps to Verify

The direct link to cause Zoovy to trigger a Paypal redirect is:
http://USERNAME.zoovy.com/paypal.cgis?orderid=YYYY-MM-#####&email=buyer@domain.com

The buyer is redirected to a Paypal screen - but before the redirect occurs it is possible to press ESCAPE in the browser at the redirect page. View the source and save it to a file on your hard disk.
View the shipping amount being sent to Paypal - it is correct.
<input type="hidden" name="shipping_1" value="28.25">
If Paypal complains about a "duplicate transaction" or how the order was already paid - you can save the redirect file to your hard disk, and use a text editor (such as textpad) to change the invoice # in the file so you can retry the same payment to Paypal.
