If your customers have Javascript disabled (Firefox: Tools: Options: Content: Enable Javascript), the evolution search does not work as intended. Our developer is going to change the wrapper so it messages your customers to this affect.

Resolution

Evolution and our other ajax themes rely heavily on javascript. When I built them, I did not design them in such a way as to be backward compatible with non-js browsers. I will, going forward, try to build our ajax applications to be more graceful when handling issues like this.
As a solution for evolution, I have added a piece of text that warns users that the site is javascript heavy. You can customize the text through the wrapper editor (site builder - edit theme) to read as you would prefer.
You can create a new profile with a subdomain that uses a non-ajax theme and link to that (using wiki) in the noscript tag so that your customers can have a good shopping experience.
