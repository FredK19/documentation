MDAC Troubleshooting Guide

Overview

Microsoft Data Access Components (MDAC) 2.8 contains core Data Access components such as the Microsoft SQL Server OLE DB provider and ODBC driver. Zoovy Order Manager uses Microsoft Access Runtime Edition as it's core database in non-enterprise installations. Unfortunately because MDAC is also relied on by numerous other applications (including Microsofts own applications) Zoovy can provide only limited support. We have verified that MDAC 2.8 installs correctly on machines with operating systems 98,ME,XP, and 2000 which are freshly installed.

This document attempts to fix the commonly known MDAC 2.8 issues:
  • Order Manager runtime error 5000
  • cp internal 103003 runtime error (kernel32 general protection fault)
  • runtime error 6 - buffer overflow
    Unfortunately once you begin receiving these errors - there is no "simple" fix.

    Attention Windows 98/ME Users - we have reason to believe that MDAC 2.8 has substantial installation and compatibility issues on Windows 98/ME. We recommend you remain with Order Manager version 4.26 or lower. You may download Order Manager version 4.26 from http://app1.zoovy.com/biz/downloads/zom_v426.exe However - once you have upgraded to 4.28, you will no longer be able to easily downgrade [thanks to Microsoft!].
    Once you have upgraded your system to MDAC 2.8 (which is installed as part of Order Manager 4.28 and higher) you will not be able to downgrade back to MDAC 2.7sp1.

    You may download MDAC 2.8 directly from Microsoft: http://www.microsoft.com/downloads/details.aspx?familyid=6c050fe3-c795-4b7d-b037-185d0506396c&displaylang=en

    We recommend all users who are experiencing difficulty with MDAC download the mdac_typ.exe file to the desktop, then run the program, this will cause MDAC to install in non-quiet mode and may provide additional insight as to what the specific problem is. 99% of the customer issues can be fixed by removing any running applications from the startup (or simply rebooting into safe mode) and then rebooting, and finally running mdac_typ.exe to install (before loading any other applications) resolves the problem.

    • Known Installation Issues Windows 95/98/ME

    The following document from Microsoft describes how to remove MDAC 2.8 from your system. http://support.microsoft.com/default.aspx?kbid=311720 .

    Refer to the Microsoft troubleshooting guide if you intend to use MDAC 2.8 installed on your system http://support.microsoft.com/default.aspx?scid=kb;EN-US;232060

    Known Installation Issues - XP/2000

    MDAC 2.8 may not install correctly if you have *ANY* Microsoft applications open. This includes any applications which you may not be aware of - to disable any microsoft applications we recommend you download the MDAC file and start your computer in safe mode, or at minimum disable any unnecessary applications from startup. To do this go to start button, click on Run, and type "msconfig" into the dialog and press "OK". Go to the startup tab and disable any applications which are not critical to the operation of the machine. Specifically Microsoft Office Search (MOSearch) is a resource hog which will also cause MDAC 2.8 to fail on installation.

    Additional troubleshooting steps, and how to contact Microsoft technical support are provided below:
    http://support.microsoft.com/default.aspx?scid=kb;EN-US;232060

    Removing MDAC 2.8

    To remove MDAC components from your PC, go to start (on the bottom left) then run and type command and hit okay:

  • Type cd \ and hit the enter key
  • Type cd progra~1 and hit enter key
  • Type cd zoovy and hit enter key
  • Type cd ordermgr and hit enter key
  • Type mdac_typ /t:c:\mdac /c and hit enter key
  • You should see the program copy files to the directory
  • Type cd \mdac and hit enter
  • Type dasetup /u and hit enter
  • Reboot the computer

    Note: Removing MDAC 2.8 can help resolve an error 5000-error on processing report in Order Manager.

    Note: If the steps above don't work, just run "dasetup.exe /u" by locating the file with a search on your hard drive. This is usually located in "C:\Program Files\Common Files\Microsoft Shared\dasetup"
