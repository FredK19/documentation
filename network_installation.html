NETWORK Suite: NETWORK Order Manager Installation

Overview

This webdoc will take you through the download and installation of the NETWORK Suite including: MySql Database, MySql GUI Administrative Tools, NETWORK Sync Manager, and NETWORK Order Manger Client.

Installation

Prerequisites

You should have a dedicated server with no other "non-server" applications installed. The server should have at least 1gb of free space, but more space is recommended. If you do plan to use this with "non-server" applications you will do the installation/setup for "non-server" instructions.
You can find out more about hardware and system requirements by reading webdoc: Doc 50650
If you are upgrading Enterprise Order Manager rather than performing the initial installation, please read: Doc 50885

Downloads

As a "pre-installation" step, there are several items you should download and save to your desktop or another area on your computer where you can retrieve them.
You can get full instructions on downloading the "pre-installation" step in the prior to Your Upgrade web doc Doc 51375.
  • ---Microsoft .NET 3.5 Framework service pack 1: You will need Microsoft .NET 3.5 framework sp1 installed on both the server and any client machines. For more information on that and other download requirements please visit: Zoovy Download Page

    Download MySql Database Scroll down the page (skip the registration) and choose a mirror that is close to you. Download the .zip file to your server machine and decompress using a program such as winzip.
  • ---MySql GUI Administrative Tools : Go to:
    Download MySql GUI Administrative Tools Under the registration box, click on the link entitled "No thanks, just take me to the downloads!". Choose a mirror that is close to you and save file to your computer.
  • ---NETWORK Sync Manager : Go to:
    The downloads page in your Zoovy account and select Sync Manager or Download NETWORK Sync Manager Select the latest version of the Zoovy Sync Manager
  • NETWORK Order Manger Client : this will need to be saved onto the machines that will be used as clients rather than the server, one copy on each machine. You can find the latest general availability release by going logging into your Zoovy Web Manager and clicking the "Downloads" link in the upper right of the screen. We always recommend that you check with Zoovy Tech Support before downloading and installing to make sure which versions are the latest recommended.

    • Step1: Install the MySQL Database

    Run the setup.exe from within the .zip file you downloaded and extracted above for MySql Database.
    Select the "Complete" install and install to C:\program files\mysql\mysql server 5.1
    If the "Sign up - Setup Wizard" screen comes up, select "Skip Sign-Up" - this is not necessary. On the "server 5.1 - Setup Wizard" screen leave the "Configure the MySQL Server Now" checkbox checked and click finish.

    Dedicate server not running "non-server" applications:

    The MySQL Server Instance Configuration Wizard 1.08 will start.
  • When prompted select the "Detailed Configuration" option - select the appropriate server type for your environment.
  • For the best performance we recommend "Dedicated MySQL Server Machine" - you should NOT choose "Developer Machine" or your performance will suffer.
  • Next you will want to select "Multifunctional Database" option
  • During the next step you will want to select the "optimal" number of connections the database will support, for most installations the default "Decision Support (DSS)/OLAP" option will be fine. If you plan to have more than 20 concurrent administrative users then please consult with Zoovy Support.
  • Leave the Enable TCP/IP Networking option with port number 3306 enable, and also leave the Enable Strict mode option selected.
  • When asked to choose a character set, please select "Manual Selected Default Character Set / Collation" and choose "UTF8"
  • Leave the "Install as Windows Service checked", along with the Launch the MySQL Server automatically option. Also check the Include Bin Directory in Windows PATH box and click the "Next >" option
  • Leave the Modify Security Settings box checked. Enter a root password, we recommend at least 8 digits using a combination of mixed case and both numbers and letters. DO NOT check the enable root access from remote machines or create an anonymous account option as these will substantially lower the security of your configuration and are not necessary.
    Remember the password, you will need it in step 2.
    When you have configured all the steps above correctly then click the "Execute" button. Once the configuration has been processed click the "Finish" button.

    • Server running "non-server" applications (if you are using the machine as a work station, not just as a server):

  • When prompted select the "Detailed Configuration" option - select the appropriate server type for your environment.
  • For the best performance we recommend "Developer Machine" - This will allow help prevent database issues when running "non-server" applications
  • Next you will want to select "Non-transactional Database" option
  • During the next step you will want to select the "optimal" number of connections the database will support, for most installations the default "Decision Support (DSS)/OLAP" option will be fine. If you plan to have more than 20 concurrent administrative users then please consult with Zoovy Support.
  • Leave the Enable TCP/IP Networking option with port number 3306 enable, and also leave the Enable Strict mode option selected.
  • When asked to choose a character set, please select "Manual Selected Default Character Set / Collation" and choose "UTF8"
  • Leave the "Install as Windows Service checked", along with the Launch the MySQL Server automatically option. Also check the Include Bin Directory in Windows PATH box and click the "Next >" option
  • Leave the Modify Security Settings box checked. Enter a root password, we recommend at least 8 digits using a combination of mixed case and both numbers and letters. DO NOT check the enable root access from remote machines or create an anonymous account option as these will substantially lower the security of your configuration and are not necessary.
    If you are not sure about any of this you can have Zoovy support install this by scheduling a GEEK services appointment in the support area.

    • Step2: Install the MySql GUI Administrative Tools

    The MySql GUI Administrative Tools are a user interface that allows the user to easily run queries against their database and export these results to a file. This is an enormously useful tool in troubleshooting any issues you may have for Zoovy Tech Support.
    Zoovy Tech Support will guide you through any actually use of these tools, but having them installed and ready will facilitate quicker resolution of any issues you may encounter.
    To install, run the installer file that you downloaded above, using the default settings.

    Step3: Install the MySql .net/Connector 5.2.5

    See Doc 50885 for the link to download the connector.
    ZID 8.x comes with the new MySQL .net/Connector in the download page under MySQL community server.

    Under the registration box, click on the link entitled "No thanks, just take me to the downloads!". Choose a mirror that is close to you and save file to your computer.

    Step4: Install the Zoovy Sync Manager

    Run the installation package for Sync Manager you downloaded above. Install using the default settings. The first time Sync Manager runs, it will request the account name and database "root" password, this is the password you created in Step 1 above. The sync manager will then ask for your Zoovy username and Master Password for the account security so it can download all your users from zoovy.com. Each user will only be able to have one concurrent connection to the database.
    If this is your first MySQL database with Zoovy, depending on how big your database is, could take several hours.

    Once SyncMgr is running you should see a 60 minute timer counting down to the next scheduled sync time, and a button allowing you to manually start the sync immediately if desired. You can start your initial sync to load the new MySQL database with all your order information at this time. This may take a while depending on the amount of data that needs to me transferred. You will also notice an "Admin" menu at the top of the window. Nearly all of Order Manager's configuration settings and preferences for Email, UPS, FedEx, Endicia and other settings have been moved centrally into the SyncMgr. You can either set these up now or after you have completed Step 3 below. As a final step before moving on to installation of the Order Manager NETWORK Client, while you are still at the server machine, you will need to open a port through the Windows Firewall if you have it running to allow the client to access the Sync Mgr and MySql database we have just installed. To do this, follow the steps below:
  • Open Windows Control Panel (Start > Control Panel)
  • Open Windows Firewall
  • Click on the "Exceptions" tab and then click on the "Add Port" button In the "Port number" field enter "3306"
  • Make sure option "TCP" is selected, and click "Ok"

    • Step5: Install the Zoovy Order Manager NETWORK Client

    Remote Clients
    Our SysOp team is able to configure remote clients via VPN (Virtual Private Network) if you have processing being done from separate networks and/or locations. This service comes at no added cost, though configuration by a SysOp is billable. Please refer to the Hamachi Webdoc for further information:
    Go to Zoovy Download Page and download and install the latest version of the Order Manager.
    After installation, start the client. At the login screen go to Admin > Database Config and choose the "NETWORK" tab. You will see three fields on this screen. The first is your Zoovy account name, the second is the IP Address for the server machine and the third is the database type with two radio buttons to choose from. One radio button will be for MySQL community Server (recommended) and the other will be Microsoft Server 2008. Click the radio button next to the database your using.
    You can obtain the server's IP Address if you don't know it by opening a command line window on the server and typing "ipconfig" (Start > Run > Type "cmd" [Enter] > Type "ipconfig" [Enter]).
    Once you have entered these items you can hit "Save" to bring you back to the login screen.
    At the login screen you will then enter your account name, user name, and user password to login and open the client. For more information and details on this process or how to setup a user, visit: Doc 50713.
