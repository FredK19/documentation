Setup

This document is intended for use by a FedEx Customer Integration Consultant (CIC). While you may be able to configure this without the assistance of a FedEx CIC - Zoovy does not recommend this.
Zoovy Order Manager creates CTS export files formatted for use with the FedEx Passport feature of Ship Manager Cafe.
The FedEx "Secret" Daily Password
ShipManager Cafe does not have the required Passport functionality by default. So as part of the installation you must enable Passport in your ShipManager Cafe software. Since FedEx considers this configuration of Passport to be too technical for your average user, you're going to need the secret "Daily" password. Fortunately, the password isn't that secret - in fact it can be easily obtained by calling FedEx ShipManager Automation Support, who will give you a password that is good for one day.

First call 1-800-GO-FEDEX and say "FedEx ShipManager Automation Support" at the first menu prompt when digital-attendant asks what service you'd like. Then select 1 for Ship Manager Support, then select 1 again for Installation Issues. Simply tell the friendly FedEx representative you wish to enable the Passport functionality and you need the password. The representative will likely ask you for your system id, this is NOT your FedEx account number - it is the meter number which is in the menu bar of your Cafe software (hint: it is a six digit number adjacent to the version number of the software). If in doubt, the FedEx Tech can probably help you find it. Once you have the password, then you may proceed with the steps below. PLEASE NOTE - FEDEX PASSWORDS ARE ALWAYS IN UPPER CASE, if you type the password in lower case it won't work! (we mention it here only because FedEx techs love to forget to mention this little detail)

On a final note: Please don't call Zoovy and ask what the ShipManager password is, we don't know - it changes every day, and we don't have it.
First we need to do is configure Order Manager to create Passport files.
'Configuring Order Manager to export FedEx Passport files'
  • Load the Zoovy Order Manager
  • Go to Tools | FedEx ShipManager Setup
  • Verify that your FedEx account # appears in the screen, if it does not then enter it.
  • Meter number can be left blank (it is not currently used)
  • Select the Default Service - this is the FedEx service which will be most commonly used. (NOTE: Zoovy will automatically flip between Home Delivery and FedEx ground for residential / non-residential addresses)
  • Select the Default Package Type - typically this will be customer packaging.
  • Specify the FedEx Passport File, we recommend you set this to C:\zoovy-passport.in (note: you must end the file with a .in extension)
  • Click the okay button to save.
    • Then all we do is configure the PassPort functionality in the FedEx Ship Manager Cafe software. (If you are doing this without a CIC and didn't read the hint above about the FedEx Daily Password, we recommend you stop here and go back and read it!)
    'Enabling Passport in FedEx Ship Manager'
  • Select Customize from the menu along the top
  • Select System Settings
  • Highlight your system profile
  • Click the modify button
  • Select tab #4 "FedEx Express Admin"
  • At this point you will be prompted for the daily password - instructions for getting the password are above. (this password changes every day, or we'd give it to you here)
  • At this point you should enable the three checkboxes for Passport System. Specifically "Batch", "Auto Listening", and "Label Transfer"
  • PLEASE - [NO MATTER HOW MUCH YOU THINK IT WILL HELP] - DO NOT MAKE ANY OTHER CHANGES TO YOUR SHIP MANAGER SOFTWARE WITHOUT CONSULTING A FEDEX CIC FIRST!
  • Now cross your fingers and click the okay button to save.

    • Usage

    Getting setup is a bit tricky, fortunately using the Passport integration on a day to day basis isn't. Just follow the simple steps below.
    'Shipping with FedEx Ship Manager'
  • In Order Manager select one or more orders and then right click and select Export to FedEx Ship Manager.
  • The C:\zoovy-passport.in file will be automatically overwritten with the new data.
  • Load the FedEx Ship Manager window (if you already have it open, thats fine too)
  • Click on "Passport" in the menu area.
  • We recommend you enable the "Batch Edit" in the processing mode area (this will let you verify the weight, and shipping type of each package)
  • Set the input file to C:\zoovy-passport.in
  • Set the export file to C:\passport-export.in
  • NOTE: While doing a test run you may want to click the Test Mode, this will prevent any real airbills from being printed.
  • Click the "Start Input" button to start reading from the input file
  • Each airbill will be displayed, you will probably want to verify the weight of each package as well as the service type.
  • NOTE: If you are using FedEx Ground as your default service type then the system will automatically check the residential box and select FedEx home delivery for any order which does not contain a company name in the ship to field.

    • Mapping Fields

    Fields that you can mapped to for reports/email include:

  • Field 1204 - Ship Alert Email ID 1 - max length 35
  • Field 3002 - Ground Invoice Number (35) - order number
  • Field 25 - Express Reference Info (35) - order number

    • Troubleshooting

    At this time we are aware that this integration does not support International Addresses. This is probably fine, we don't think any of our customers ship internationally with FedEx, if you plan to - please let us know and we'll add it.

    If you receive a parsing error please make a note of the error and open a Zoovy support ticket and MAKE SURE you attach the zoovy-passport.in file to the ticket. Please also include the version # of the Cafe software and if there was more than one transaction see if you can identify which order # it occurred on. (HINT: the zoovy-passport.in file gets overwritten each time you export a new order, so don't wait until your done processing orders.)

    If you receive a "Runtime Error 76" exporting to Fedex Ship Manager, please check your configuration to make sure your mapping is correct. If you are mapping to a network drive, make sure the drive is mounted.

    This is currently an experimental feature, you're welcome to use it, and we think it works great - but please be aware that any fixes will likely require a new release of the Order Manager client and could take a week [or longer] to implement. We are actively seeking comments on how we can enhance this feature.
