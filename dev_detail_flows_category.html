Page Title

Sometimes it is more convenient to have a page title in the flow, instead of the wrapper. To do this, place the following piece of code into the flow.
<ELEMENT TYPE="READONLY" NAME="TITLE"> LOAD=FLOW::TITLE </ELEMENT>

Default Product List:

<ELEMENT TYPE="PRODLIST" NAME="PRODLIST1"> PRETTY=Product Listing ABOUT=You can add products to your homepage, when you do they will appear below. DEFAULT=&FORMAT=THUMB&COLS=3&ALTERNATE=0 SAVETO=page:prodlist </ELEMENT>
Also, you can add a SRC=NAVCAT:.safe.page.path to reference another pages prodlist which gives you the ability to have multiple product lists on a single page*. safe.page.path is the safe page name/path of the prodlist you are referring to. For instance, if you had a category shoes with a subcategory of red, the URL would look like: http://YOURUSERNAME.zoovy.com/category/shoes.red the path would be .shoes.red
*If you want to use the LIST features, do not specify a source.

Custom Product Listing Style Image Manipulation

In a product listing style, you can set various attributes of the images being used. For instance, the height, width, background color (padding), and the alt tag.
W: Sets the maximum width of the image.
H: Sets the maximum height of the image.
M Minimal mode (do not pad image to full dimensions specified in the prodlist).
B: Sets the background color (and padding) of the image to specified HEX#.
ALT: Sets the alt tag of the image to the specified zoovy product field.
default- Sets the image to default to another if the image isn't specified.
For example:
%zoovy:prod_thumb/default-zoovy:prod_image1/image-w:200-h:100-m-b:FFFFFF-alt:prod_name%
This would display the thumbnail at a max. height of 100 and a max. width of 200. The 'm' means it would not pad itself to exactly 200x100 and the b sets the background color to white (FFFFFF). The alt tag on the product would display the product name.

Custom Product Listing Style Field Manipulation

In many cases, when you are creating a product listing style, you'll want a break tag or a space to appear only if the field contains data. For instance, if you have a prodlist that displays a name, description and price and you set it up as follows:
%zoovy:prod_name%<br> %zoovy:prod_desc%<br> Price: $%zoovy:base_price%
What if there was no description for one of the products? Then you would have a blank line between the product name and the price. Also, if there was no price (say it was a redirect instead of a purchasable item) then it would say 'Price: $' with nothing to follow. Instead, you could set it up as follows:
%zoovy:prod_name/noblank:break% %zoovy:prod_desc/noblank:break% %zoovy:base_price/money%
This would accomplish the same effect as the top listing style, however it would only put in the break tags if the space was not blank. Here are several other rules you can apply to the name fields:
%zoovy:foo/if-zoovy:bar% This will output the contents of zoovy:foo only if zoovy:bar IS NOT blank or 0.
%zoovy:foo/unless-zoovy:bar% This will output the contents of zoovy:foo only if zoovy:bar IS blank or 0.
%zoovy:foo/pretext-zoovy:bar% This will prepend the contents of zoovy:bar on zoovy:foo, only if zoovy:foo is not blank.
%zoovy:foo/posttext-zoovy:bar% This will postpend the contents of zoovy:bar onto zoovy:foo only if zoovy:foo is not blank.
%zoovy:base_price/subtract-field-zoovy:prod_msrp/money% This will do some math for you. It will subtract the base price from the MSRP and return the difference. It's a 'You Save' function, more or less.
%zoovy:base_price/percent-value-80/money% This is another math function that will return a percentage of the original ammount. In this case, if the items original base price is $10.00, this will return a value of 8.
Characters, spaces, and breaks By using any of of the following as seen in the example below, you can add them to a prodlist with rules associated to them. For instance:
%username:msrptext/if-zoovy:msrp/noblank-space:after%%zoovy:msrp/money/noblank-break:after%
This would add the user-specific msrptext field (such as 'retail price:' or 'msrp:') and then add a space after it, but only if there is an MSRP price set in the system for this product. If a price is set, it would add both of these two fields, followed by a break tag. If no MSRP is set, it would not add either of these two items, or the break tag.
slashWill add a '/' as specified.
barWill add a '|' as specified.
spaceWill add a ' ' as specified.
breakWill add a '<br>' as specified.
moneyWill treat the field as a dollar value and format it accordingly.
strip Will truncates and strips breaks and other HTML from the field. If no_strip_breaks is 1, it doesn't strip breaks. if no_strip_html is 1 it doesn't strip html.
hidezero If the value is set to zero, then it will set the value to zero.

SUBCAT ELEMENT

<ELEMENT TYPE="SUBCAT" NAME="SUBCAT">
MODE=
PRETEXT= text which appears at the beginning if the list is not blank
POSTTEXT= text which appears at the end if the list is not blank
DEFAULT= text which is displayed if there are no sub categories. </ELEMENT>
This element has three basic modes of operation: PLAIN, BUTTONS, LIST

MODE=PLAIN
Plain generates the default plain style, with text links, no graphics, it is simple, but not configurable.

MODE=BUTTONS
Use MODE=BUTTONS if you want to display graphic buttons to your user as part of the page flow.
In addition if you set the following in the element tag:
BUTTONINI=name of ini file -- the following optional button properties apply: -- FORCEWIDTH=, FORCEHEIGHT=

MODE=LIST
The LIST mode provides you with the ability to do single, or multi-column (but not multi-page) category lists, as well as the ability to customize virtually any aspect of the appearance of a subnav cat.

If you are already familiar with lists then you should know that the following special variables apply within <!-- CATEGORY --> tags:

cat_url = the full url (with http://) to category
cat_pretty = the pretty name
cat_safe = the safe name (without the leading period)

In addition all the standard theme variables are available, however merchant and product namespace data is not.
