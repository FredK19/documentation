ZoovyMail: How to Start

Overview

This document is intended to assist users in migrating their domain names and accounts to the ZoovyMail system.

Step 1: Create your ZoovyMail Admin Account

'Creating the Admin Mail account'
  • Login to www.zoovy.com
  • Go to Setup | Domain Hosting
  • Click to enable ZoovyMail
  • Wait approximately 5 minutes
  • Return to Domain Hosting and a password will be displayed.
    • Your Zoovymail account cannot be provisioned unless you have filled out your Company Information found under Setup > Site Builder.
    If you are having trouble creating an email alias it could be that the alias already exists under your newsletter settings. Please check what you have entered in Setup > Domain Hosting > Domain Config > Add Newsletter Alias.

    Step 2: Add Additional Users

    For each individual inbox (separate mailbox) you will need to create an employee subaccount in your Zoovy Store.
    'Creating Subusers w/individual inboxes'
  • Login to www.zoovy.com
    For Mobile users (PDA'S) use www.fusemail.com/mobile
  • Go to Setup | Manage Users
  • Add or Edit an account
  • Check the box which says "Create individual inbox for this employee"
  • Press save when done.
  • Wait 5 minutes for the inbox to be provisioned.

    • Step 3: Add Domains

    If you already have your domain name pointing at the Zoovy nameservers then you must complete this step and the next step by 6:15am PST, 12:15pm PST, 6:15pm PST, or 12:15am PST. Any changes you make to your domain names will become active and if you have not finished setting up any aliases then you could lose mail.
    'Adding Domains'
  • Login to www.zoovy.com
  • Go to Setup | Domain Hosting
  • Click the "Add/New" button
  • In the mail configuration area select: ZoovyMail 2.0 (Recommended)

    • Step 4: Setup Shared Folders, Mail Aliases, and Forwarding Rules

    The ZoovyMail platform has the ability to provide "Shared Folders" which can be seen by multiple users. We recommend setting up aliases, along with mail rules such as "orders", "support", "info", "questions", "ebay", "amazon", etc. to be routed into a Folder in your "admin@USERNAME.zoovy.com" account.
    This makes it easy to add/remove access to folders without the need to change passwords or manage separate accounts for each alias. This configuration is also the most compatible with upcoming Zoovy features which will make your employees more productive.
    'Setting up a Folder'
  • Go to http://webmail.zoovy.com
  • Login as "admin@USERNAME.zoovy.com" with the password provided in Setup | Domain Hosting
  • Click on "Settings" in the upper toolbar (note: you may be on this screen by default when you login)
  • Click on "Folder Management" in the left hand menu
  • Type in the Folder name (e.g. "info", "support", "questions", "orders")
  • Folder type should be "Mail Items"
  • Select "Root Directory" from the list of folders below.
  • Click the "Add Folder" button to save.
    Note: folders may take a few moments to appear in the list, don't worry - this is normal.
  • Repeat this process for each alias you intend to create.
    • The following assumes you are still logged into you mail Admin account at http://webmail.zoovy.com from the previous set of instructions.
    'Setting up a Mail Alias with a Mail Rule'
  • Click on "Settings" in the upper toolbar
  • Click on "Setup Mail Aliases" which appears in the center of the screen, below "Setup Consolidated Accounts"
  • Click "Add a new email alias to a domain name"
  • Enter the mail alias name e.g. "orders" and select a domain name from the list of domain names.
  • Click the "Add Mail Rule to this Mail Account"
  • Under "Rule Name and Options", enter the email address in "Rule Name" (e.g. "orders@mywherehouse.com").
  • Leave the default options checked. They include "Apply rule to: Incoming Email", "Rule scope: Local", and "Before Spam & Virus Check: Process After Spam & Virus Filter"
  • Under "Conditions to create a match", it has already been filled out for you from creating the alias. Skip this section.
  • Under "Actions to perform when a match is found", select "Move it to the specified folder" by "Select Action to add:".
  • By "Current Actions" you'll see an orange box pop up where you can select the folder into which you want your messages stored (e.g. "orders"). Click "Save" in the orange box.
  • Click "Save Rule" to add the rule.
  • All email for the alias that you defined should now automatically appear in the folder you selected.
    • The following assumes you are still logged into you mail Admin account at http://webmail.zoovy.com from the previous set of instructions.
    'Sharing a Folder with other Users'
  • Click on "Settings" in the upper toolbar
  • Select "Folder Management" from the menu on the left
  • Click the "Properties" button which appears in the submenu (adajcent to Create | Rename | Move | Delete)
  • Select the Folder you want to share, then click "View/Edit" properties
  • Select the user you want to share with and click the "Add Sharing" button
  • The shared folder will appear in the users list of folders.
    • Previous Zoovy Mail options provided wildcard forwarding (forwarding of any mail message) to a single account. In an effort to fight spam you must now configure each mail alias, no wildcard forwarding is available.

    Step 5: Point your nameservers at Zoovy

    Before proceeding with this step, be sure to verify that Zoovy's nameservers have loaded your changes. In Setup | Domain Hosting your domain name will say "Active" instead of "Pending"
    When you are ready to start receiving mail then login to the company you purchased the domain name from and change your nameservers to:
  • primary: ns.zoovy.com
  • secondary: ns2.zoovy.com

    • Step 6: Setting up Mail clients

    You can use virtually any popular email software (such as Outlook, Outlook Express, or Mozilla Thunderbird) to read and send your email. The steps for setting up each mail client are located at http://webdoc.zoovy.com/webmail

    Step 7: Setup Newsletter Aliases

    The Zoovy Newsletter feature in Zoovy will automatically handle bounced messages and unsubscribe requests. Before you can send a newsletter you will need to configure one or more newsletter sender addresses.
    'Configure Newsletter Sender Address'
  • Login to www.zoovy.com
  • Go to Setup | Domain Hosting
  • In the "Sender/From" address enter "newsletter"
  • In the "Forward Responses To", if you leave this blank then email responses to your newsletter will NOT be forwarded, if you set this then the Zoovy system will attempt to sort through bounces, and unsubscribe requests and only forward you the relevant emails.
  • If you configure an autoresponse text then that message will be sent. Some people prefer to respond with a short and simple message such as "To contact us please email questions@domain.com"
